# EchoIM 未来优化规划

本文档记录 EchoIM 后续可优化方向，按优先级和领域分类。

---

## 一、工程化

### 1.1 CI/CD 流水线

当前状态：无自动化构建和部署流程。

优化方向：
- GitHub Actions 配置 lint + typecheck + test + build 流水线
- 自动部署到测试环境
- 生产环境手动触发部署

### 1.2 E2E 测试

当前状态：无端到端测试。

优化方向：
- Playwright 测试覆盖核心流程：登录、单聊收发、群聊收发、消息撤回编辑、通话流程
- 测试用例放在 `frontend/echoim-web/e2e/`

### 1.3 后端单元测试

当前状态：有少量 Service 层测试骨架。

优化方向：
- AuthService、ConversationService、MessageCommandService 核心逻辑覆盖
- Mock Mapper 层，专注业务逻辑验证

### 1.4 API 版本管理

当前状态：所有 API 在 `/api/` 下，无版本前缀。

优化方向：
- 迁移到 `/api/v1/` 前缀
- 为后续 API 演进做准备

## 二、性能优化

### 2.1 消息搜索性能

当前状态：全局搜索使用 `LIKE` 全表扫描，数据量大时性能差。

优化方向：
- MySQL FULLTEXT INDEX 或接入 Elasticsearch
- 消息内容分词索引
- 搜索结果缓存

### 2.2 数据库分页优化

当前状态：消息列表使用 offset 分页，深分页性能差。

优化方向：
- 已有 cursor 分页（`selectMessageCursorByConversationIdAndUserId`），推广到更多场景
- 会话列表也改为 cursor 分页

### 2.3 会话列表性能

当前状态：会话列表查询每次关联多张表。

优化方向：
- Redis 缓存会话列表（按用户）
- 消息写入时异步更新缓存
- 未读计数完全基于 Redis 原子操作

### 2.4 文件上传体验

当前状态：文件上传无进度反馈。

优化方向：
- 前端 XMLHttpRequest 监听 upload.onprogress
- 大文件分片上传
- 断点续传

### 2.5 限流升级

当前状态：`LocalRateLimitService` 基于 JVM 内存（ConcurrentHashMap），多实例不共享。

优化方向：
- 基于 Redis 的分布式限流（令牌桶/滑动窗口）
- 支持 IP 级别 + 用户级别双重限流

## 三、功能增强

### 3.1 PWA 支持

当前状态：纯 Web 应用，无离线能力。

优化方向：
- Service Worker 缓存静态资源
- 消息队列离线暂存，上线后自动发送
- Web Push 通知（Push API + Notification API）
- manifest.json 配置，安装到桌面

### 3.2 国际化 (i18n)

当前状态：所有文案硬编码中文。

优化方向：
- vue-i18n 接入
- 提取所有中文文案到语言包
- 补充英文翻译
- 语言切换 UI

### 3.3 消息加密

当前状态：消息明文存储，传输依赖 HTTPS/WSS。

优化方向：
- 端到端加密（E2EE）：Signal Protocol 或 MTProto 方案
- 密钥交换：基于 Diffie-Hellman
- 本地密钥存储：IndexedDB + 用户密码保护
- 影响范围：消息搜索、服务端内容审核需适配

### 3.4 多端同步增强

当前状态：前端支持多账号切换，但同一账号多设备在线时消息同步依赖离线补发。

优化方向：
- 设备间消息实时同步（多设备同时在线）
- 已读状态跨设备同步
- 输入状态跨设备感知

### 3.5 Webhook 与开放接口

当前状态：系统封闭，无外部集成能力。

优化方向：
- Webhook 回调：消息事件、群组变更、用户事件
- 开放 API 文档（OpenAPI 3.0）
- API Key 认证方式

### 3.6 系统公告推送增强

当前状态：公告存储在数据库，前端未主动拉取。

优化方向：
- 用户端登录后拉取未读公告
- WebSocket 推送新公告
- 公告弹窗或消息形式展示

## 四、安全加固

### 4.1 分布式会话管理

当前状态：JWT 无状态，注销/强制下线依赖 Redis 标记。

优化方向：
- Refresh Token 存入 Redis，支持主动注销
- Token 黑名单机制
- 设备级别的 Token 管理

### 4.2 审计日志增强

当前状态：`sys_operation_log` 表已存在，但只有管理后台操作记录。

优化方向：
- 用户端关键操作也记录审计日志
- 日志导出（CSV/Excel）
- 日志保留策略（自动清理 N 天前）

### 4.3 文件安全增强

当前状态：Tika 内容检测 + 扩展名白名单。

优化方向：
- 病毒扫描集成（ClamAV）
- 图片 EXIF 信息清除
- 文件访问权限控制（签名 URL 时效）

---

## 五、运维与可观测性

### 5.1 日志收集

当前状态：应用日志输出到 stdout，无集中收集。

优化方向：
- ELK（Elasticsearch + Logstash + Kibana）或 Loki + Grafana
- 结构化日志格式（JSON）
- 日志级别动态调整

### 5.2 监控告警

当前状态：有健康检查和基础指标端点，无告警。

优化方向：
- Prometheus + Grafana 指标采集与可视化
- 关键指标告警：在线用户数骤降、消息发送失败率升高、WS 连接数异常
- 钉钉/飞书/邮件告警通知

### 5.3 链路追踪增强

当前状态：MDC 注入 traceId，慢请求告警（>3s）。

优化方向：
- 接入 SkyWalking 或 Jaeger
- 跨服务链路追踪（如果后续拆分微服务）
- 请求耗时分析

---

## 六、移动端

### 6.1 移动端适配

当前状态：响应式布局，但非原生体验。

优化方向：
- 方案 A：PWA（轻量，复用现有代码）
- 方案 B：React Native / Flutter 原生应用
- 方案 C：Capacitor / Ionic 混合应用

### 6.2 推送通知

当前状态：仅浏览器 Notification API，需页面打开。

优化方向：
- FCM（Android）/ APNs（iOS）推送
- 需要配合移动端应用

## 优先级建议

| 优先级 | 方向 | 原因 |
|--------|------|------|
| P0 | 系统公告推送增强 | 功能已建好但前端未接入 |
| P0 | CI/CD 流水线 | 基础工程化保障 |
| P1 | PWA + Web Push | 离线消息触达 |
| P1 | 消息搜索性能 | 数据量增长后的瓶颈 |
| P1 | E2E 测试 | 质量保障 |
| P2 | 国际化 | 出海准备 |
| P2 | 分布式限流 | 多实例部署需要 |
| P2 | 监控告警 | 运维保障 |
| P3 | 消息加密 | 高安全场景 |
| P3 | 移动端 | 扩展用户群 |
