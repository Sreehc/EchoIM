# EchoIM 接口文档

## 1. 文档说明

本文档用于定义 EchoIM 的接口分组、请求响应规范和 WebSocket 协议约定，供前后端联调使用。

接口分为两类：

- HTTP 接口：用于登录、联系人、群组、会话、文件和后台管理等业务
- WebSocket/Netty 接口：用于实时消息、心跳、消息确认、离线补发和系统通知

---

## 2. 基础约定

### 2.1 HTTP 服务前缀

- 用户端：`/api`
- 管理端：`/admin`

### 2.2 鉴权方式

- HTTP：`Authorization: Bearer {token}`
- WebSocket：握手时携带 token，或首次发送 `AUTH` 消息完成认证

### 2.3 响应格式

统一响应格式：

```json
{
  "code": 0,
  "message": "success",
  "data": {},
  "requestId": "202604221230001234"
}
```

分页接口响应格式：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "list": [],
    "pageNo": 1,
    "pageSize": 20,
    "total": 0
  },
  "requestId": "202604221230001234"
}
```

### 2.4 通用错误码建议

| code | 含义 |
|---|---|
| 0 | 成功 |
| 40000 | 参数错误 |
| 40100 | 未登录或 token 无效 |
| 40300 | 无权限 |
| 40400 | 资源不存在 |
| 40900 | 业务冲突 |
| 50000 | 系统异常 |

## 3. HTTP 接口

## 3.1 认证接口

### 3.1.1 注册

- 方法：`POST /api/auth/register`

请求示例：

```json
{
  "username": "echo_demo_01",
  "password": "123456",
  "nickname": "Echo用户01"
}
```

返回字段建议：

```json
{
  "userId": 10001,
  "username": "echo_demo_01",
  "nickname": "Echo用户01"
}
```

### 3.1.2 登录

- 方法：`POST /api/auth/login`

请求示例：

```json
{
  "username": "echo_demo_01",
  "password": "123456"
}
```

返回字段建议：

```json
{
  "token": "jwt-token",
  "tokenType": "Bearer",
  "expiresIn": 7200,
  "userInfo": {
    "userId": 10001,
    "username": "echo_demo_01",
    "nickname": "Echo用户01",
    "avatarUrl": null
  }
}
```

### 3.1.3 退出登录

- 方法：`POST /api/auth/logout`

### 3.1.4 修改密码

- 方法：`POST /api/auth/change-password`

请求示例：

```json
{
  "oldPassword": "123456",
  "newPassword": "12345678"
}
```

## 3.2 用户接口

### 3.2.1 获取当前用户信息

- 方法：`GET /api/users/me`

### 3.2.2 修改当前用户信息

- 方法：`PUT /api/users/me`

请求示例：

```json
{
  "nickname": "新昵称",
  "avatarUrl": "https://cdn.example.com/avatar.png",
  "gender": 1,
  "signature": "hello echoim"
}
```

### 3.2.3 搜索用户

- 方法：`GET /api/users/search?keyword=echo&pageNo=1&pageSize=20`
- 说明：按 `userNo / username / nickname` 搜索正常状态用户，不返回当前登录用户，`pageSize` 最大为 `100`

返回示例：

```json
{
  "list": [
    {
      "userId": 10002,
      "userNo": "E10002",
      "username": "echo_demo_02",
      "nickname": "Echo用户02",
      "avatarUrl": null,
      "gender": 0,
      "signature": "hello echoim",
      "friendStatus": "PENDING_OUT",
      "pendingRequestId": 50001
    }
  ],
  "pageNo": 1,
  "pageSize": 20,
  "total": 1
}
```

`friendStatus` 取值：

- `SELF`：当前登录用户本人
- `BLOCKED_OUT`：我已拉黑对方
- `BLOCKED_IN`：对方已拉黑我
- `FRIEND`：已是好友
- `PENDING_OUT`：我已向对方发起申请，待处理
- `PENDING_IN`：对方已向我发起申请，待处理
- `NONE`：无好友关系和待处理申请

### 3.2.4 获取他人公开资料

- 方法：`GET /api/users/{id}`
- 说明：返回公开资料和当前登录用户与目标用户的好友状态，不返回 `phone / email`

返回示例：

```json
{
  "userId": 10002,
  "userNo": "E10002",
  "username": "echo_demo_02",
  "nickname": "Echo用户02",
  "avatarUrl": null,
  "gender": 0,
  "signature": "hello echoim",
  "status": 1,
  "friendStatus": "FRIEND",
  "pendingRequestId": null
}
```

## 3.3 联系人接口

### 3.3.1 获取好友列表

- 方法：`GET /api/friends`

### 3.3.2 发起好友申请

- 方法：`POST /api/friend-requests`

请求示例：

```json
{
  "toUserId": 10002,
  "applyMsg": "你好，想加你为好友"
}
```

### 3.3.3 获取好友申请列表

- 方法：`GET /api/friend-requests`

### 3.3.4 同意好友申请

- 方法：`PUT /api/friend-requests/{id}/approve`

### 3.3.5 拒绝好友申请

- 方法：`PUT /api/friend-requests/{id}/reject`

### 3.3.6 删除好友

- 方法：`DELETE /api/friends/{friendId}`
- 说明：删除后保留单聊会话和历史消息，但双方不能继续互发消息

### 3.3.7 修改好友备注

- 方法：`PUT /api/friends/{friendId}/remark`

请求示例：

```json
{
  "remark": "产品同学"
}
```

### 3.3.8 获取黑名单

- 方法：`GET /api/friends/blocked`

### 3.3.9 拉黑好友

- 方法：`PUT /api/friends/{friendId}/block`
- 说明：单向拉黑，任一方向存在拉黑关系时，双方都不能继续发送单聊消息

### 3.3.10 解除拉黑

- 方法：`PUT /api/friends/{friendId}/unblock`

## 3.5 会话接口

### 3.5.1 获取会话列表

- 方法：`GET /api/conversations?pageNo=1&pageSize=20`

返回字段建议：

```json
{
  "list": [
    {
      "conversationId": 30001,
      "conversationType": 1,
      "conversationName": "Echo用户02",
      "avatarUrl": null,
      "lastMessagePreview": "hello",
      "lastMessageTime": "2026-04-22 21:00:00",
      "unreadCount": 2,
      "isTop": 0,
      "latestSeq": 125
    }
  ],
  "pageNo": 1,
  "pageSize": 20,
  "total": 1
}
```

### 3.5.2 获取历史消息

- 方法：`GET /api/conversations/{id}/messages?pageNo=1&pageSize=20`
- 上拉历史：`GET /api/conversations/{id}/messages?maxSeqNo=120&pageSize=20`
- 单会话补拉：`GET /api/conversations/{id}/messages?afterSeq=120&pageSize=100`
- `maxSeqNo` 和 `afterSeq` 不能同时传；`maxSeqNo` 查询更早消息，`afterSeq` 查询本地未同步的新消息。

### 3.5.3 置顶会话

- 方法：`PUT /api/conversations/{id}/top`

请求示例：

```json
{
  "isTop": 1
}
```

### 3.5.4 删除会话

- 方法：`DELETE /api/conversations/{id}`

### 3.5.5 标记已读

- 方法：`PUT /api/conversations/{id}/read`

请求示例：

```json
{
  "lastReadSeq": 18
}
```

## 3.6 离线同步接口

### 3.6.1 全局离线消息补拉

- 方法：`POST /api/offline-sync/messages`
- 说明：客户端重连后上报本地每个会话最后同步到的 `lastSyncSeq`，服务端一次返回当前用户所有有效会话中未同步的消息。
- 注意：补拉只解决“未同步”，不会自动标记已读，也不会清空未读数。

请求示例：

```json
{
  "defaultLastSyncSeq": 0,
  "syncPoints": [
    {
      "conversationId": 30001,
      "lastSyncSeq": 120
    }
  ],
  "perConversationLimit": 50,
  "totalLimit": 500
}
```

返回示例：

```json
{
  "conversations": [
    {
      "conversation": {
        "conversationId": 30001,
        "conversationType": 1,
        "conversationName": "10001_10002",
        "lastMessagePreview": "hello",
        "lastMessageTime": "2026-04-23T10:00:00",
        "unreadCount": 3,
        "isTop": 0,
        "latestSeq": 125
      },
      "fromSeq": 121,
      "toSeq": 125,
      "hasMore": false,
      "messages": []
    }
  ],
  "hasMore": false
}
```

## 3.7 文件接口

### 3.7.1 上传文件

- 方法：`POST /api/files/upload`
- 内容类型：`multipart/form-data`
- 说明：要求登录，后端中转上传阿里云 OSS，再写入 `im_file`
- 表单参数：
  - `file`：二进制文件
  - `bizType`：可选，`2` 图片，`4` 普通文件；不传时按 MIME 自动判断

返回示例：

```json
{
  "fileId": 50001,
  "fileName": "demo.png",
  "fileExt": "png",
  "contentType": "image/png",
  "fileSize": 102400,
  "bizType": 2,
  "objectKey": "echoim/2026/04/24/10001/abc123.png",
  "downloadUrl": "https://bucket.oss-cn-shanghai.aliyuncs.com/...",
  "expiresIn": 600,
  "expireAt": "2026-04-24T12:00:00"
}
```

### 3.7.2 获取文件信息

- 方法：`GET /api/files/{id}`
- 说明：要求登录；上传者本人或该文件所属消息会话成员可访问

### 3.7.3 下载文件

- 方法：`GET /api/files/{id}/download`
- 说明：要求登录；返回临时签名下载链接

返回示例：

```json
{
  "fileId": 50001,
  "downloadUrl": "https://bucket.oss-cn-shanghai.aliyuncs.com/...",
  "expiresIn": 600,
  "expireAt": "2026-04-24T12:00:00"
}
```

## 4. WebSocket/Netty 协议

## 4.1 消息类型

- `AUTH`：连接认证
- `PING`：心跳请求
- `PONG`：心跳响应
- `CHAT_SINGLE`：单聊消息
- `CHAT_GROUP`：群聊消息
- `ACK`：送达确认
- `READ`：已读确认
- `NOTICE`：系统通知
- `CONVERSATION_CHANGE`：会话列表变更通知
- `FORCE_OFFLINE`：强制下线
- `OFFLINE_SYNC`：离线消息同步

## 4.2 通用报文格式

```json
{
  "type": "CHAT_SINGLE",
  "traceId": "trace-001",
  "clientMsgId": "cmsg-123456",
  "timestamp": 1713772800000,
  "data": {}
}
```

字段说明：

| 字段 | 类型 | 说明 |
|---|---|---|
| type | string | 消息类型 |
| traceId | string | 请求链路追踪 ID |
| clientMsgId | string | 客户端消息 ID，用于幂等 |
| timestamp | long | 客户端时间戳 |
| data | object | 业务载荷 |

## 4.3 认证消息

客户端发送：

```json
{
  "type": "AUTH",
  "timestamp": 1713772800000,
  "data": {
    "token": "jwt-token"
  }
}
```

## 4.4 心跳消息

客户端发送：

```json
{
  "type": "PING",
  "timestamp": 1713772800000,
  "data": {}
}
```

服务端响应：

```json
{
  "type": "PONG",
  "timestamp": 1713772800001,
  "data": {}
}
```

```json
  "type": "CHAT_SINGLE",
```

```json

  "type": "ACK",

    "serverMsgId": 90000001,
```

客户端发送：

```json
{
  "type": "READ",
  "timestamp": 1713772801000,
  "data": {
    "conversationId": 30001,
    "lastReadSeq": 18
  }
}
```

## 4.9 离线同步

离线消息同步以 HTTP `POST /api/offline-sync/messages` 为准。`OFFLINE_SYNC` 类型保留给后续服务端主动推送能力，本阶段不使用。

## 4.10 会话变更通知

服务端推送：

```json
{
  "type": "CONVERSATION_CHANGE",
  "timestamp": 1713772803000,
  "data": {
    "changeType": "MESSAGE_NEW",
    "conversation": {
      "conversationId": 30001,
      "conversationType": 1,
      "conversationName": "10001_10002",
      "lastMessagePreview": "hello",
      "lastMessageTime": "2026-04-23T10:00:00",
      "unreadCount": 1,
      "isTop": 0,
      "latestSeq": 125
    },
    "message": {
      "messageId": 40001,
      "conversationId": 30001,
      "seqNo": 125,
      "fromUserId": 10001,
      "toUserId": 10002,
      "msgType": "TEXT",
      "content": "hello",
      "sentAt": "2026-04-23T10:00:00"
    }
  }
}
```

`changeType` 当前支持：

- `MESSAGE_NEW`：新消息导致会话摘要、最新序号或未读数变化
- `READ_UPDATE`：已读操作导致当前用户未读数变化
- `CONVERSATION_CREATED`：好友申请通过后创建或恢复单聊会话

## 5. 关键流程说明

## 5.1 单聊消息流程

1. 客户端生成 `clientMsgId` 并发送消息。
2. 服务端校验 token、好友关系和发送权限。
3. 服务端创建或查找单聊会话。
4. 服务端为当前会话分配递增 `seqNo`。
5. 消息持久化到 `im_message`。
6. 若接收方在线，立即推送消息。
7. 更新双方会话摘要和未读数。
8. 返回 ACK 给发送方。
9. 接收方发送 READ 回执后更新已读状态。

## 5.2 群聊消息流程

1. 客户端发送群聊消息。
2. 服务端校验群成员身份和群状态。
3. 持久化群消息。
4. 广播给在线群成员。
5. 批量更新群成员会话未读数。
6. 记录送达或已读回执。

## 5.3 离线与重连策略

- 客户端重连后立即调用 `POST /api/offline-sync/messages`。
- 客户端按会话维护本地最后同步到的 `lastSyncSeq`。
- 服务端返回各会话 `seqNo > lastSyncSeq` 的未同步消息。
- 离线补拉不等于已读，不会自动推进 `lastReadSeq`。
- 对重复 `clientMsgId` 做幂等去重。

## 5.4 文件消息策略

- 先上传文件拿到 `fileId`。
- 再发送文件类型消息。
- 消息体只引用资源，不保存二进制。
- 图片和普通文件统一走 `im_file` 表。
- 文件由后端中转上传阿里云 OSS。
- 接收方按需用临时签名 URL 下载到本地缓存或用户指定位置。

## 5.5 集群扩展策略

- Redis 记录用户所在实例。
- 同实例直接推送。
- 跨实例通过 Redis Pub/Sub 或 Redisson topic 分发。
- 连接断开时清理在线状态和路由缓存。
