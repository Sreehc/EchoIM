# EchoIM 接口文档

## 1. 文档说明

本文档用于定义 EchoIM 的接口分组、请求响应规范和 WebSocket 协议约定，供前后端联调使用。

接口分为两类：

- HTTP 接口：用于登录、联系人、群组、会话、文件和后台管理等业务
- WebSocket/Netty 接口：用于实时消息、心跳、消息确认、离线补发和系统通知

---

## 2. 基础约定

### 2.1 HTTP 服务前缀

- 用户端：`/api`
- 管理端：`/admin`

### 2.2 鉴权方式

- HTTP：`Authorization: Bearer {token}`
- WebSocket：握手时携带 token，或首次发送 `AUTH` 消息完成认证

### 2.3 响应格式

统一响应格式：

```json
{
  "code": 0,
  "message": "success",
  "data": {},
  "requestId": "202604221230001234"
}
```

分页接口响应格式：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "list": [],
    "pageNo": 1,
    "pageSize": 20,
    "total": 0
  },
  "requestId": "202604221230001234"
}
```

### 2.4 通用错误码建议

| code | 含义 |
|---|---|
| 0 | 成功 |
| 40000 | 参数错误 |
| 40100 | 未登录或 token 无效 |
| 40300 | 无权限 |
| 40400 | 资源不存在 |
| 40900 | 业务冲突 |
| 50000 | 系统异常 |

## 3. HTTP 接口

## 3.1 认证接口

### 3.1.1 注册

- 方法：`POST /api/auth/register`

请求示例：

```json
{
  "username": "echo_demo_01",
  "password": "123456",
  "nickname": "Echo用户01"
}
```

返回字段建议：

```json
{
  "userId": 10001,
  "username": "echo_demo_01",
  "nickname": "Echo用户01"
}
```

### 3.1.2 登录

- 方法：`POST /api/auth/login`

请求示例：

```json
{
  "username": "echo_demo_01",
  "password": "123456"
}
```

返回字段建议：

```json
{
  "token": "jwt-token",
  "tokenType": "Bearer",
  "expiresIn": 7200,
  "userInfo": {
    "userId": 10001,
    "username": "echo_demo_01",
    "nickname": "Echo用户01",
    "avatarUrl": null
  }
}
```

### 3.1.3 退出登录

- 方法：`POST /api/auth/logout`

### 3.1.4 修改密码

- 方法：`POST /api/auth/change-password`

请求示例：

```json
{
  "oldPassword": "123456",
  "newPassword": "12345678"
}
```

## 4. WebSocket/Netty 协议

## 4.1 消息类型

- `AUTH`：连接认证
- `PING`：心跳请求
- `PONG`：心跳响应
- `CHAT_SINGLE`：单聊消息
- `CHAT_GROUP`：群聊消息
- `ACK`：送达确认
- `READ`：已读确认
- `NOTICE`：系统通知
- `FORCE_OFFLINE`：强制下线
- `OFFLINE_SYNC`：离线消息同步

## 4.2 通用报文格式

```json
{
  "type": "CHAT_SINGLE",
  "traceId": "trace-001",
  "clientMsgId": "cmsg-123456",
  "timestamp": 1713772800000,
  "data": {}
}
```

字段说明：

| 字段 | 类型 | 说明 |
|---|---|---|
| type | string | 消息类型 |
| traceId | string | 请求链路追踪 ID |
| clientMsgId | string | 客户端消息 ID，用于幂等 |
| timestamp | long | 客户端时间戳 |
| data | object | 业务载荷 |

## 4.3 认证消息

客户端发送：

```json
{
  "type": "AUTH",
  "timestamp": 1713772800000,
  "data": {
    "token": "jwt-token"
  }
}
```

## 4.4 心跳消息

客户端发送：

```json
{
  "type": "PING",
  "timestamp": 1713772800000,
  "data": {}
}
```

服务端响应：

```json
{
  "type": "PONG",
  "timestamp": 1713772800001,
  "data": {}
}
```

## 4.5 单聊消息

客户端发送：

```json
{
  "type": "CHAT_SINGLE",
  "traceId": "trace-001",
  "clientMsgId": "cmsg-123456",
  "timestamp": 1713772800000,
  "data": {
    "conversationId": 30001,
    "toUserId": 10002,
    "msgType": "TEXT",
    "content": "hello"
  }
}
```

## 4.6 群聊消息

客户端发送：

```json
{
  "type": "CHAT_GROUP",
  "traceId": "trace-002",
  "clientMsgId": "cmsg-123457",
  "timestamp": 1713772800100,
  "data": {
    "conversationId": 30002,
    "groupId": 20001,
    "msgType": "TEXT",
    "content": "大家晚上好"
  }
}
```

## 4.7 ACK 回执

服务端响应：

```json
{
  "type": "ACK",
  "traceId": "trace-001",
  "clientMsgId": "cmsg-123456",
  "timestamp": 1713772800020,
  "data": {
    "serverMsgId": 90000001,
    "conversationId": 30001,
    "status": "DELIVERED"
  }
}
```

## 4.8 已读回执

客户端发送：

```json
{
  "type": "READ",
  "timestamp": 1713772801000,
  "data": {
    "conversationId": 30001,
    "lastReadSeq": 18
  }
}
```

客户端发送：

```json

  "type": "OFFLINE_SYNC",

  "data": {
    "conversationId": 30001,

  }
}
```

## 5. 关键流程说明

## 5.1 单聊消息流程

1. 客户端生成 `clientMsgId` 并发送消息。
2. 服务端校验 token、好友关系和发送权限。
3. 服务端创建或查找单聊会话。
4. 服务端为当前会话分配递增 `seqNo`。
5. 消息持久化到 `im_message`。
6. 若接收方在线，立即推送消息。
7. 更新双方会话摘要和未读数。
8. 返回 ACK 给发送方。
9. 接收方发送 READ 回执后更新已读状态。

## 5.2 群聊消息流程

1. 客户端发送群聊消息。
2. 服务端校验群成员身份和群状态。
3. 持久化群消息。
4. 广播给在线群成员。
5. 批量更新群成员会话未读数。
6. 记录送达或已读回执。

## 5.3 离线与重连策略

- 客户端重连后立即触发离线同步。
- 以 `lastReadSeq` 或最后拉取时间为补偿游标。
- 服务端返回游标之后的新消息。
- 对重复 `clientMsgId` 做幂等去重。

## 5.4 文件消息策略

- 先上传文件拿到 `fileId`。
- 再发送文件类型消息。
- 消息体只引用资源，不保存二进制。
- 图片、视频和普通文件统一走 `im_file` 表。

## 5.5 集群扩展策略

- Redis 记录用户所在实例。
- 同实例直接推送。
- 跨实例通过 Redis Pub/Sub 或 Redisson topic 分发。
- 连接断开时清理在线状态和路由缓存。
